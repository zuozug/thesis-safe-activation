# 第四章 实验设计与结果分析

## 4.1 实验环境

本文实验基于 Python 与 PyTorch 框架实现。实验对象为 MNIST 手写数字数据集，优化器采用 Adam，损失函数采用 CrossEntropyLoss。训练与评估过程中，重点记录以下内容：

- 训练损失
- 验证准确率
- 测试准确率
- 训练总时间
- 推理时间

后续正式成稿时，应补充具体的软件版本、硬件配置、batch size、epoch 和学习率等详细参数。

## 4.2 基线模型设计

本文采用一个用于 MNIST 分类的小型卷积神经网络作为实验对象。其结构大致如下：

1. 第一层卷积；
2. 第一层激活；
3. 第一层池化；
4. 第二层卷积；
5. 第二层激活；
6. 第二层池化；
7. 展平；
8. 全连接层；
9. 激活；
10. 输出层。

该模型既可以作为 baseline，也可以通过模块封装或自动替换方式构造为近似激活版本。

## 4.3 函数级实验设计

函数级实验用于评估单个函数近似的误差表现。对每个函数，均输出以下内容：

- 原函数曲线
- 近似函数曲线
- 绝对误差曲线
- MAE
- MSE
- Max Error

建议配置如下：

- ReLU：2 / 4 / 6 阶，区间 `[-3, 3]`
- Sigmoid：3 / 5 / 7 阶，区间 `[-4, 4]`
- GELU：3 / 5 / 7 阶，区间 `[-3, 3]`
- Softmax：指数近似 + 稳定归一化

## 4.4 模块级实验设计

模块级实验的主要目的是验证模块设计本身的正确性，而非单纯考察模型精度。验证点包括：

- 模块是否能够正常前向传播；
- 自动替换是否成功；
- 输出 shape 是否保持一致；
- 模块接口是否统一；
- Softmax 近似输出是否接近概率分布。

对应结果可通过单元测试、替换记录、前向输出示例等形式展示。

## 4.5 模型级对照实验设计

本文模型级实验分为主实验与补充实验两类。

### 4.5.1 主实验

- Baseline CNN
- ApproxReLU CNN
- ApproxGELU CNN

### 4.5.2 补充实验

- ApproxSigmoid CNN
- Softmax 输出层轻量实验

### 4.5.3 模型级指标

模型级指标主要包括：

- Accuracy

- Training Time
- Inference Time

## 4.6 实验结果与分析

本节在正式实验完成后，应从以下几个角度进行分析：

1. 不同函数的函数级近似误差对比；
2. 不同激活替代方案对模型精度的影响；
3. 替代方案对训练时间和推理时间的影响；
4. 不同阶数和不同区间下的消融结果；
5. Softmax 近似的稳定性与概率分布性质。

### 4.6.1 ReLU 主实验分析

重点比较 Baseline CNN 与 ApproxReLU CNN 的训练损失曲线、验证准确率曲线、测试准确率和时间开销。

### 4.6.2 GELU 主实验分析

重点比较 ApproxGELU CNN 与 Baseline CNN 的性能差异，并说明更复杂激活函数替代时的误差与开销平衡。

### 4.6.3 Sigmoid 补充实验分析

作为补充实验，Sigmoid 更侧重说明平滑激活函数近似时的效果，不要求像主实验一样展开全部对照。

### 4.6.4 Softmax 补充实验分析

Softmax 重点分析：

- 分类准确率是否发生明显变化；
- 概率分布误差是否可接受；
- 概率和是否接近 1；
- 是否出现数值不稳定现象。
