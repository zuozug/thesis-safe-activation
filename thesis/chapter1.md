# 第一章 绪论

## 1.1 研究背景与意义

随着云计算、大模型服务和机器学习即服务（MLaaS）的快速发展，越来越多的模型训练与推理任务被部署到云端完成。云端部署降低了使用门槛，但也带来了新的隐私风险：用户往往需要将原始数据上传到第三方平台，在医疗、金融、政务等敏感场景下，这种方式可能导致严重的数据泄露风险。

为了解决“想利用云端模型能力，但又不希望泄露原始数据”的矛盾，学术界提出了同态加密（HE）和多方安全计算（MPC）等隐私保护技术。它们的共同特点是能够在不暴露输入内容的前提下完成计算过程，因此非常适合隐私保护推理场景。

然而，HE 和 MPC 对神经网络中的复杂非线性运算支持较弱。相比加法和乘法，比较、指数、除法、归一化等操作在加密环境下开销高、实现复杂，而 ReLU、Sigmoid、GELU、Softmax 等常用非线性函数恰恰依赖这些运算。这使得非线性层成为隐私保护神经网络推理中的关键瓶颈。

因此，研究如何使用多项式近似等更适合加密环境的形式替代原始非线性函数，具有重要意义。一方面，这有助于降低密态推理的计算复杂度；另一方面，也能够为后续更复杂模型的隐私兼容改造提供模块化基础。

## 1.2 国内外研究现状

近年来，围绕“如何用低阶多项式替代神经网络中的非线性函数”这一问题，国内外已经开展了大量研究。现有研究主要集中在以下几个方向：

1. 使用泰勒展开、最小二乘拟合、Chebyshev 多项式等方法构造低阶近似；
2. 使用分段多项式或分段线性函数提升局部精度；
3. 使用查表法（LUT）降低单次计算复杂度；
4. 针对 Softmax 等高复杂度函数设计专门的稳定化近似方案；
5. 从 HE/FHE 兼容性角度重新设计激活函数。

总体来看，现有研究已经证明：多项式近似是连接“神经网络非线性需求”和“加密环境低阶算术约束”的有效桥梁。但不同函数的逼近难点差异明显，例如 ReLU 的尖点结构、Sigmoid 的饱和区、GELU 的高阶非线性以及 Softmax 的归一化结构，都需要分别考虑。

## 1.3 本文研究内容

本文围绕“用于模型非线性函数替代的安全算法模块设计”这一主题，开展如下研究：

1. 分析 ReLU、Sigmoid、GELU、Softmax 四类函数的数学特征及其在 HE/MPC 场景下的实现难点；
2. 设计基于多项式近似的统一替代接口，提供 `f(x)` 与 `approx_f(x)` 两类函数；
3. 将替代函数封装为可插拔的 PyTorch 模块，并实现模型激活层自动替换；
4. 以 MNIST 小型 CNN 为实验对象，对比 baseline 模型与替代模型在精度、损失、训练时间、推理时间等方面的表现；
5. 通过函数级、模块级、模型级和消融实验分析不同近似方案的效果。

## 1.4 本文结构安排

全文共分为五章：

- 第一章介绍研究背景、意义、国内外研究现状以及本文研究内容；
- 第二章介绍 HE/MPC 对神经网络计算的限制、多项式逼近原理和四类函数的数学特征；
- 第三章重点介绍非线性函数替代模块的设计，包括统一接口、模块封装和自动替换机制；
- 第四章给出实验设计、实验结果与分析；
- 第五章总结全文工作，并对后续改进方向进行展望。
